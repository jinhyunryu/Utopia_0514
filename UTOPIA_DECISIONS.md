# UTOPIA_DECISIONS
# 사용자 승인 결정 기록

> 이 문서는 사용자가 직접 승인한 개발/설계 결정만 기록한다.  
> 단, 이 문서에 기록되었다고 해서 자동으로 원본 LOCK이 되는 것은 아니다.  
> 실제 LOCK은 `UTOPIA_CORE.md`, `UTOPIA_DATA.md`, `UTOPIA_DESIGN.md`, `UTOPIA_APPENDIX.md`에 반영된 뒤 확정된다.  
> 이 문서의 목적은 “왜 이 결정을 했는지”와 “어떤 원본 문서에 반영해야 하는지”를 추적하는 것이다.

---

## 0) 상태 태그

- **APPROVED**: 사용자가 승인했지만 원본 반영 전
- **APPLIED**: 원본 문서 반영 완료
- **SUPERSEDED**: 이후 결정으로 대체됨
- **REVOKED**: 사용자 판단으로 승인 철회됨

---

## 1) 작성 원칙

- 사용자의 명시적인 승인 없이 항목을 추가하지 않는다.
- Codex가 임의로 결정을 확정하지 않는다.
- 결정 항목에는 반드시 영향 범위를 적는다.
- LOCK 변경이면 사유와 영향 문서를 적는다.
- 원본 문서 반영 후에는 `CHANGELOG.md`에도 실제 변경 이력을 남긴다.
- 과거 결정과 충돌하는 경우 `SUPERSEDED` 처리하고 대체 결정을 연결한다.


## 3) 승인 결정 목록

### 2026-05-14 / 예시 — 개발 중 수정 후보와 승인 결정을 분리 관리

- 상태: APPROVED
- 승인자: 사용자
- 관련 CHANGE_QUEUE 항목:
  - 없음
- 결정 내용:
  - 개발 중 떠오르는 수정 후보는 `UTOPIA_CHANGE_QUEUE.md`에 보관한다.
  - 사용자가 승인한 결정은 `UTOPIA_DECISIONS.md`에 별도로 기록한다.
  - 실제 원본 반영 후에는 `CHANGELOG.md`에 변경 이력을 남긴다.
- 결정 사유:
  - 개발 중 발생하는 수정 아이디어가 원본 LOCK 문서에 바로 섞이지 않도록 하기 위함.
  - 최종 수정 판단은 사용자만 수행하도록 분리하기 위함.
- LOCK 변경 여부:
  - 없음. 문서 운영 방식에 대한 결정이다.
- 영향 범위:
  - 문서 운영
  - Codex 작업 방식
  - 변경 이력 관리
- 반영 대상 문서:
  - CORE:
  - DATA:
  - DESIGN:
  - APPENDIX:
  - CHANGELOG:
    - 필요 시 문서 운영 방식 변경 이력으로 기록 가능
- 반영 상태:
  - 대기
- 후속 작업:
  - 필요 시 `README.md`에 문서 운영 방식으로 추가
- 비고:
  - 원본 설정 내용 자체는 변경하지 않는다.

## 4) 결정 처리 흐름

```text
UTOPIA_CHANGE_QUEUE.md의 항목 검토
↓
사용자 승인
↓
UTOPIA_DECISIONS.md에 APPROVED로 기록
↓
원본 문서 반영 여부 판단
↓
원본 반영 완료
↓
상태를 APPLIED로 변경
↓
CHANGELOG.md에 실제 변경 이력 기록
```

---

## 5) Codex 사용 규칙

Codex는 이 파일을 사용할 때 다음 규칙을 따른다.

- 이 파일에 없는 내용을 승인된 결정으로 취급하지 않는다.
- 사용자의 명시 승인 없이 항목을 추가하지 않는다.
- 승인된 결정이라도 원본 문서에 반영되기 전까지는 LOCK으로 취급하지 않는다.
- 원본 문서에 반영된 항목만 `APPLIED`로 변경한다.
- 반영 완료 시 `CHANGELOG.md` 갱신 필요 여부를 사용자에게 확인한다.
