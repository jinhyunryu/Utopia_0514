# UTOPIA_CHANGE_QUEUE
# 개발 중 수정 후보 / 검토 대기 목록

> 이 문서는 원본 설정이 아니다.  
> 여기에 적힌 내용은 확정(LOCK)이 아니며, 사용자의 명시 승인 전까지 `UTOPIA_CORE.md`, `UTOPIA_DATA.md`, `UTOPIA_DESIGN.md`, `UTOPIA_APPENDIX.md`에 반영하지 않는다.  
> Codex는 이 파일을 근거로 원본 문서를 자동 수정하지 않는다.  
> 이 파일의 목적은 개발 중 발생한 수정 아이디어, 충돌 의심점, 개선 후보를 안전하게 보관하는 것이다.

---

## 0) 상태 태그

- **IDEA**: 개발 중 떠오른 생각
- **NEED_REVIEW**: 사용자 검토 필요
- **APPROVED**: 사용자 승인 완료
- **REJECTED**: 사용자 판단으로 폐기
- **APPLIED**: 원본 문서 반영 완료
- **BLOCKED**: LOCK 충돌 또는 근거 부족으로 보류

---

## 1) 작성 원칙

- 이 문서에 적힌 내용은 확정(LOCK)이 아니다.
- 원본 문서와 충돌하는 내용은 자동 반영하지 않는다.
- Codex가 개발 중 발견한 개선점은 이 문서에 `IDEA` 또는 `NEED_REVIEW` 상태로만 추가한다.
- 사용자의 명시 승인 없이 `APPROVED`로 바꾸지 않는다.
- `APPROVED` 상태가 되더라도 원본 반영 전까지는 여전히 대기 항목이다.
- 실제 원본 문서에 반영된 뒤에만 `APPLIED`로 변경한다.
- LOCK 변경 가능성이 있는 항목은 반드시 영향 범위를 적는다.
- 고유명, 수치, 표, 세부 설정은 원문 근거가 없으면 확정하지 않는다.

## 3) 개발 중 수정 후보 목록

### 2026-05-14 / 예시 — 전투 UI에 브레이크 예고 표시 추가

- 상태: IDEA
- 분야: UI / 전투
- 발생 위치: 전투 HUD 구현 중
- 관련 원문:
  - CORE: C4 전투 핵심 / 브레이크 규칙
  - DATA:
  - DESIGN:
  - APPENDIX:
- 현재 문제:
  - 플레이어가 다음 공격으로 브레이크가 발생하는지 직관적으로 알기 어렵다.
- 제안 수정:
  - 공격 대상 선택 시 예상 브레이크 감소량과 브레이크 발생 여부를 미리 표시한다.
- LOCK 충돌 가능성:
  - 낮음. 기존 브레이크 수치 자체를 변경하지 않고 표시만 추가한다.
- 영향 범위:
  - 전투 HUD
  - 스킬 툴팁
  - 타겟 프리뷰
- 사용자 판단:
  - 미결정
- 반영 대상 문서:
  - 승인 시 `UTOPIA_DESIGN.md` 또는 UI 관련 섹션
- 비고:
  - 수치 변경 없음.

## 4) 처리 흐름

```text
개발 중 발견
↓
UTOPIA_CHANGE_QUEUE.md에 IDEA / NEED_REVIEW로 기록
↓
사용자 판단
↓
APPROVED 또는 REJECTED
↓
APPROVED 항목만 원본 문서 반영 검토
↓
원본 반영 완료 후 APPLIED로 변경
↓
CHANGELOG.md에 실제 반영 이력 기록
```

---

## 5) Codex 사용 규칙

Codex는 이 파일을 사용할 때 다음 규칙을 따른다.

- 이 파일의 항목을 LOCK으로 취급하지 않는다.
- 이 파일의 항목만 보고 원본 문서를 수정하지 않는다.
- 사용자 승인 문장이 없는 항목은 `APPROVED`로 바꾸지 않는다.
- 개발 중 발견한 수정 후보는 이 파일에만 추가한다.
- 원본 문서 수정이 필요한 경우, 먼저 사용자에게 승인 여부를 묻는다.
- 승인된 항목을 원본에 반영했다면 `UTOPIA_DECISIONS.md`와 `CHANGELOG.md` 갱신 필요 여부를 함께 확인한다.
